Checkout

  • CheckoutButton

    This is the main point of interaction for your customers. The user will tap this button in order to initiate the Visa Checkout SDK’s user interface. A CheckoutButton can have a CheckoutButtonStyle for different visual rendering options.

    You can use the onCheckout(total:currency:completion:) methods to provide information that is used when the button is tapped by the user.

    Alternatively, you may add a target-action mechanism provided by UIControl instances. You may then call the VisaCheckoutSDK.checkout() methods directly to manually launch the Visa Checkout SDK’s user interface.

    CheckoutButton can only have 2 heights - 34 or 47 points. Setting the height < 34 will result in a height of 34 points. Setting the height > 47 will result in a height of 47. If the height is set somewhere between 34-40, the height will snap down to 34. If the height is set somewhere between 41-47, the height will snap up to 47. CheckoutButton width can be anywhere from 154 to 425. Widths < 154 will snap to 154 and widths > 425 will snap to 425. However, unlike height, the width can be set anywhere between 154-425 and that width will be honored.

    Example setting the purchase information to an instance of CheckoutButton*

    checkoutButton.onCheckout(total: 22.09, currency: .usd) { result in
   print(result.statusCode)
}

  • CheckoutResult

    The CheckoutResult class is the object that is returned when a Visa Checkout launch has occurred and finished. This class will contain information related to the user’s payment if successful. If the checkout attempt was unsuccessful, it will contain an error code in statusCode.
